# Руководство по участию в разработке

## Процесс разработки

1. Форкните репозиторий
2. Создайте feature ветку
3. Сделайте коммиты
4. Откройте Pull Request

## Стандарты кода

- Используйте TypeScript для фронтенда
- Следуйте Google Go Style Guide для бэкенда
- Пишите комментарии для сложной логики
- Тестируйте свой код

## Коммит-сообщения

Используйте conventional commits:
- `feat:` для новой функциональности
- `fix:` для исправления ошибок
- `docs:` для документации
- `style:` для форматирования
